Build your blog by Hugo + Github Pages
A simple tutorial to help you build your blog website use hugo and github.
2023-01-31 13:05 folder [tutorials]   label Hugo   blog  

0 Foreword

This is my first blog and I want to record the process I build this blog site on Windows.

1 Prerequisites

First, you need to download/install the following:

Go

Git

Hugo

Please add them to environment variables.

You can check using the following commands:

go version
git version
hugo version

2 Build your site

2.1 generate folder

Run the following command:

hugo new site my_blog

It would generate a folder named my_blog in current directory and the following are the functions of these files/folders:

  • config.toml : the configuration file.

  • content : stores all the content of the website, including all of your blog posts, resumes and so on, can include folders, but usually all .md files.

  • static : stores static files such as your background pictures, logos, css, js, etc. The files in this directory will be directly copied to it /public. This folder has higher priority than the /staticfolder under the theme.

  • themes : stores all the themes downloaded and you can choose the theme by editing config file.

  • archetypes : stored .mdtemplate files, it has higher priority than /archetypesfolders under the theme.

  • data : stores data files for template calls.

  • layouts : store .htmltemplates, this folder has higher priority than /layoutsfolders under the theme.

  • public : After executing the hugocommand, store the generated static file.

2.2 download and use theme

You can go to this site to find themes. My blog uses diary so I’ll take it as an example.

cd my_blog
cd themes
git clone https://github.com/AmazingRise/hugo-theme-diary

These commands would download the theme into themes folder. To use it, you can open config.toml, and add:

theme = 'diary'

Save it, and run:

hugo server

Open http://127.0.0.1:1313, you can see that the theme is used.

2.3 write an article
cd my_blog
hugo new posts/post1.md

It would generate a file called post1.md. Open it, set draft to be false. Below that, you can write your article using Markdown.

Save it and run the server, you can see your article now!

3 Deploy your site on Github Pages

  1. Firstly, you should register a github account.

  2. Open your home page, click “+”, and then click “new repository”. The name should be like “username.github.io”, then click “create repository”.

  3. cd my_blog
hugo
cd public 
git init
git add .
git remote add origin https://github.com/username/username.github.io.git
git commit -m "first commit"
git push origin master

    These commands would build your site and send them to github.

    4. Open your repository, click “settings”, open “Pages”, click the link and that’s your website!

If you want to update your blog, run the following commands:

cd my_blog
hugo
cd public 
git init
git add .
git commit -m "update"
git push origin master
